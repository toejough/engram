We prototyped isolating each Relay shard's sync work in its own spawned subprocess, so the OS could reclaim a shard's memory whenever its subprocess exited or recycled. Peak per-shard memory did shrink, but supervising a subprocess per shard — launching, IPC, lifecycle — added complexity disproportionate to the memory saved. Rejected. Revisit only if a lighter in-process route (pooling or releasing buffers) proves insufficient.
