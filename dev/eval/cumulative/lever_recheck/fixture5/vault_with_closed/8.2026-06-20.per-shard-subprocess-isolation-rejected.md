We piloted per-shard subprocess isolation for Relay on one shard class — each shard's sync in its own spawned process, so the OS could reclaim the shard's memory at process exit. During the daily bulk-resync, the fan-in IPC serialization tripled sync latency, and peak memory moved to the coordinator process, which buffers every shard's fan-in — the cgroup limit was still crossed, just in the coordinator instead of the shards. Abandoned.
