We tried shrinking the archive's page size (the chunk size written to object storage) to cut Ledgerline's cost, expecting fewer bytes read per query. Total pipeline cost changed by well under 1%, because archive reads are served through a cache layer billed at a flat per-request rate, independent of page size — and the migration churned the write path for two weeks. Rolled back. Not worth it.
