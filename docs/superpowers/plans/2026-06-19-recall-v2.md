**Goal:** Restructure engram recall into two channels — a clustered **relevance** channel (recency-biased cosine matches → within-cluster `candidate_l2s`) and an un-clustered **recency** channel ("what we've been up to") — and collapse the L2 tier vocabulary to "notes".

**Architecture:** All changes live in the `engram query --synthesize-l2` path (the only recall mode) + `recall/SKILL.md`. The matched set is bounded to ~300 (10 phrases × top-30) so clustering stays ≈+0.2s; recent chunks are appended **un-clustered** so they're cheap and large. L2→notes is a mechanical rename bundled in.

**Tech stack:** Go; `targ` for ALL build/test/check (NO `targ build` target — install via `go install ./cmd/engram`); tests imptest + rapid + gomega. Verify with the real installed binary + timed queries (vault note 54).

## Design: the two channels

**Channel 1 — Relevance (clustered):**
1. Per phrase, rank **notes + chunks together** by **recency-biased cosine** (notes: best-of situation/body × note-recency decay, as `rankCandidates` does; chunks: cosine × `applyChunkRecency`). Take top-`matchPhraseLimit` (30).
2. Union across the 10 phrases, dedup keeping max score; drop items below `recencyBiasedFloor`; cap at `matchSetCap`. → **matched set** (≤300).
3. Cluster ONLY the matched set (`clusterUnionForSynthesis`, ≈+0.2s) → within-cluster top-`candidateL2K` (5) `candidate_l2s` (Phase 4).

**Channel 2 — Recency (un-clustered):**
4. Append the `recentFillChunks` newest chunks by `IngestedAt` (reuse `newestChunkItems`/`maxTurnBySource` ordering), tagged as a recent block, NOT added to the clustering subgraph.

**Return / activation:** `items[]` = all matched (≤300, by score) + the recent block. **Activate every returned note** (Phase 5). Quality filtering is the `0.25` floor on matches, applied *before* return — never by capping.
