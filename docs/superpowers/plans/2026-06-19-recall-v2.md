L2→notes is a mechanical rename bundled in.

**Tech stack:** Go; `targ` for ALL build/test/check (NO `targ build` target — install via `go install ./cmd/engram`); tests imptest + rapid + gomega. Verify with the real installed binary + timed queries (vault note 54).

## Design: the two channels

**Channel 1 — Relevance (clustered — this IS the D1 clustering):**
1. **Per-phrase unified ranking (resolves the Gate-A Phase-2 gap):** in `unionDirectHits`'s existing per-phrase loop (`query.go:2322`), for each phrase embed once, then rank BOTH notes (`rankCandidates`: best-of situation/body × note-recency decay) AND chunks (cosine × `applyChunkRecency`) against that phrase vector into one combined `[]scoredItem` list; take top-`matchPhraseLimit` (30). This requires per-phrase chunk scores, which `scoreChunks` does NOT produce today (it returns one best-across-phrases score). Implement a per-phrase chunk scorer (score the loaded chunk records against the single phrase vector) used inside the loop — do NOT reuse the global `scoreChunks`.
2. Union across the 10 phrases, dedup keeping max score; drop items below `matchRelevanceFloor` (on baseScore); cap at `matchSetCap`. → **matched set** (≤300).
3. Cluster ONLY the matched set (`clusterUnionForSynthesis`, ≈+0.2s) → within-cluster top-`candidateL2K` (5) `candidate_l2s` (Phase 4).

**Channel 2 — Recency (un-clustered):**
4. Append the `recentFillChunks` newest chunks by `IngestedAt` (reuse `newestChunkItems`/`maxTurnBySource` ordering), deduped against the matched set, tagged `recent`, NOT added to the clustering subgraph.

**Return / activation:** `items[]` = all matched (≤300, by recency-biased rank) + the recent block. **Activation is agent-driven, tracking USE not return:** the binary emits NO `activated` flag; the skill calls `engram activate` on only the notes the agent actually drew on (the `candidate_l2s` judged Covered/Near at Step 2.5; notes cited at Step 3). A returned-but-unused note's `LastUsed` goes stale → it fades by recency rank — bumping every returned note would defeat the supersede-by-competition mechanism (DECISION-1). Chunks are never activated. Quality filtering is the `matchRelevanceFloor` on matches, before return — never by capping.
