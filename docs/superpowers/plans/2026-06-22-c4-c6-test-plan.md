# C4–C6 Test Plan — standards-change, recent-history, emergent-synthesis

> **For agentic workers:** this is a TEST-DESIGN deliverable (a committed plan). It specifies live
> cold-vs-warm exercises for the three eval axes the cumulative CRUD eval and the tactical traps do
> NOT exercise. No exercises are built or run this pass (spec only).

**Goal:** Design cheap, reproducible live tests for **C4** (memory adapts when a standard changes
over time / supersession), **C5** (memory surfaces recent history the agent should honor), **C6**
(memory drives emergent synthesis — a new integrative note from a cluster). Each follows the
trap-harness pattern proven this session: a triggering condition, arms (cold = no memory; warm =
memory present), a deterministic-or-judge check, a cold-confirm-first gate, and results in a
labeled-criteria table.

**Context — what is and isn't measured so far:**
- **C3 (fewer interactions): measured and defensible** — cumulative eval (~−15–25% convention
  restatement, p<0.001) + the 5 tactical traps (see "trap flip" below).
- **C1 (faster) / C2 (cheaper): NOT defensible** — noise-dominated at n=20, retracted per the
  2026-06-21 retro. Do not treat as measured.
- **C4/C5/C6: not exercised live.** Each already has **scoring machinery + unit fixtures wired into
  `validate.py`** (zero-LLM checks) — they are NOT yet wired as live model-running exercises. This
  plan closes that gap; it does not reinvent the scorers.

**Trap flip (the C3 anchor this plan builds on):** 5 tactical traps, each a distinct local
convention. Cold opus committed every trap (5/5 trials each → 25/25 trials trap); warm opus (one
seeded note per convention, recall fires) applied every one (25/25 trials pass the deterministic
check). See `dev/eval/traps/RESULTS.md`.

**Verdict vocabularies differ by mode (do not conflate):**

- **Live mode** (`stub=False`, paid, separate-model adversarial, majority of `JUDGE_RUNS=3`,
  defaults `NOT_SYNTHESIS`): `SYNTHESIS` / `NOT_SYNTHESIS`. The live tests below use **live mode**.

## Global constraints (load-bearing, from memory)

- **The signal must come from MEMORY, never the prompt.** The superseding standard (C4), the recent
  decision (C5), and the synthesis-enabling cluster (C6) live in the vault/chunk index — the task
  prompt must NOT state them. Otherwise the test measures prompt-following, not memory. Single most
  important design rule.
- **Cold-confirm first (saturation gate).** Before warm spend, run the cold control n=5: the check
  must reliably show the un-helped behavior. If cold already passes unaided, the axis is saturated
  on that case — drop/redesign. (Per the opus oracle-saturation finding + the trap protocol.)
- **C5 must test recency, NOT relevance** (vault notes 42/44, 34): a recent item among *homogeneous
  old* distractors only proves the recency-band fires. Spread distractor ages (0.5d/2d/5d/15d/60d),
  and the planted recent item R must be topically *distant* from the build task so it surfaces by
  recency, not cosine.
- **Results in a labeled criteria table with units** — columns mapped to C1–C6, arms side by side,
  Δ column, axes-not-exercised marked.
- Spec only this pass; deterministic checks preferred (C4, C5) over judge (C6).

## C4 — Standards change over time (supersession)

**Claim under test:** when a convention is established and later *changed*, memory carries the
**current** standard so the agent adopts X′ and drops X — without being told in the prompt.

**Convention:** error handling (anchors to the `wrapped-error` trap; cold opus defaults to bare
`return err`). X = "wrap with `%w`", X′ = "use sentinels, X is superseded" (from `reversal_spec.json`).

**Three arms** (note: ALL warm arms carry memory; the difference is *what* is in the vault):

| arm | vault contents | expected `score_supersession` |
|---|---|---|
| **cold** (no memory) | empty | neither (opus default: bare `return err`) — `follows_x=false, follows_x_prime=false` |
| **warm-X** | only the OLD note (X: wrap with %w) | `follows_x` (agent wraps) |
| **warm-X+X′** | X PLUS a newer superseding note (X′: use sentinels, supersedes X) | **`supersession_correct`** (follows_x_prime, not follows_x) |

**Task prompt (no convention mentioned):** "Implement `Load(path string) ([]byte, error)` that reads
a file and returns its bytes, returning any error to the caller." (Same shape as `wrapped-error`.)

**Check (deterministic):** `score_supersession(generated_code, reversal_spec)`.
- **C4 metric = supersession rate**: fraction of warm-X+X′ trials where `supersession_correct`.
- **The flip that proves C4:** warm-X `follows_x` → warm-X+X′ `supersession_correct`. Memory didn't
  just install a convention (C3) — it *reversed* an installed one when the standard changed.

**Saturation gate (tests the cold CONTROL):** the cold arm must reliably NOT produce
`supersession_correct` (opus doesn't spontaneously adopt sentinels). AND warm-X+X′'s sentinel
behavior must NOT occur in the cold arm — if opus defaults to sentinels unaided, the X→X′ pair is
saturated; pick an X′ that contradicts opus's prior.

**Cost:** trap-class, ~$0.30–0.50/trial warm. n=5 × 3 arms.

## C5 — Recent history surfaced and honored

**Claim under test:** a decision made *recently* (in-session, not a crystallized lesson) surfaces in
recall's recent channel and the agent honors it — even when it isn't the most topically-relevant
item.

**Condition:** plant a recent chunk **R** stating an arbitrary in-session decision. `R_basename` =
the chunk's source identifier as it appears in the recall payload's recent channel (the
`source#anchor` / note-basename `recent_channel_surfaced` matches on). R states, e.g.: *"Decision
(just now): in this project, indent with 2 spaces and prefix every helper with a `// NOTE:` line."*
Ingest R as recent activity; surround with age-spread distractor chunks (0.5d/2d/5d/15d/60d) on
adjacent topics. **R must be topically distant from the build task** (so it surfaces by recency,
not cosine).

| arm | vault/chunks | expected |
|---|---|---|
| **cold** (R absent) | distractors only, no R | agent uses its default (tabs/gofmt, no NOTE markers) |
| **warm** (R present, recent) | R in recent channel + distractors | R surfaces; agent applies 2-space + `// NOTE:` |

**Task prompt (decision NOT mentioned):** exactly — "Add an exported function `Helper()` to utils.go
that returns the current Unix timestamp as a string. Include a short comment above it." (Multiple
sites where indentation and the comment-marker decision visibly apply.)

**Check (two named stages, both deterministic):**
1. **C5a surfaced:** `score_recency_hit(recall_payload, R_basename).target_surfaced == True` — did
   the recent channel carry R? Isolates the recency *mechanism*.
2. **C5b honored:** grep the generated code — pass iff (a) indentation is 2-space (not tab) on the
   function body AND (b) a `// NOTE:` marker comment is present. **Pass condition: both, on the
   function this task adds** (single site — no threshold ambiguity).

**Validity gate (NAMED check, not a floating caveat) — C5-validity:** run a variant where R is
topically *distant* from the build task's query terms (it already is, by construction). Assert
`target_surfaced == True` even though R is NOT a top-cosine match. Concretely: the build task is
about a timestamp helper; R is about indentation/markers — low cosine overlap. If R only surfaces
when made topically adjacent, the test is measuring relevance, not recency (the note-42/44 failure)
— FAIL the validity gate and redesign.

**Cost:** ~$0.30–0.50/trial warm. n=5 × 2 arms + the validity variant.

## C6 — Emergent synthesis (compounding lessons)

**Claim under test:** when the vault holds a *cluster* of related notes that individually don't
state an integrative principle, recall crystallizes a NEW note Z that synthesizes them (not a
restatement) — and does NOT fire when the principle is already covered.

**Condition (fixtures exist):** per `synthesis_fixtures/fixture{1,2,3}/`, each has
`vault_with_cluster/` (cluster present, no integrative note), `vault_covered/` (principle already
noted), `task.txt` (the build task), `expected_synthesis_z.txt` (what Z should integrate).

| arm | vault | expected |
|---|---|---|
| **warm-cluster** | `vault_with_cluster/` | agent runs `/recall` → crystallizes Z (a NEW note) |
| **control-covered** | `vault_covered/` | agent runs `/recall` → does NOT crystallize (already covered) |

**Task prompt (triggers recall against the fixture vault):** use each fixture's `task.txt` verbatim,
prefixed with the recall-first instruction (same as the trap warm runner: "Before writing any code,
invoke your /recall skill and apply everything relevant. Then: <task.txt>").

**Check (two stages):**
1. **C6a fired (deterministic):** did `/recall` write a NEW note? Snapshot vault note count
   (`*.md`) before vs after the run; `crystallized = (count_after > count_before)`. Capture the new
   note's content for stage 2. (Mirrors the cumulative harness's `notes_written` delta.)
2. **C6b quality (live adversarial judge):** `judge_crystallization(vault_path, task_txt,
   expected_z_txt, stub=False, crystallized_note_content=<new note>, judge_model=...)` →
   `SYNTHESIS` (Z integrates the cluster, not a restatement) vs `NOT_SYNTHESIS`.

- **C6 metric = SYNTHESIS rate on warm-cluster AND MUST-NOT-FIRE rate on control-covered** (both
  matter — a judge/skill that always synthesizes is useless; the covered arm must show
  `crystallized == False` or a NOT_SYNTHESIS no-op).

**Cost:** higher — recall + crystallization + 3 judge calls/trial. n=5 × (3 fixtures × 2 arms).

## Interpretation guidance (read AFTER the designs — not a ranking of effort)

All three are specified to the same build-readiness. They differ in *what they measure*, which
matters when reading results — not in how much rigor to invest:

- **C4** measures the model's *code output* deterministically — directly comparable to C3, and it
  captures real downstream task value (did the agent adopt the current standard?).
- **C5** has a partly-mechanical half (C5a surfaced is by-construction) and a value half (C5b
  honored — did the agent *act* on recent-but-not-most-relevant memory). Weight C5b.
- **C6** measures the `/recall` skill's *capture judgment*, not downstream build quality. A
  SYNTHESIS verdict means "recall wrote a good integrative note," NOT "memory made the model build
  better." Report it as such.
