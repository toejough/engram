# C4–C6 Test Plan — standards-change, recent-history, emergent-synthesis

> **For agentic workers:** this is a TEST-DESIGN deliverable (a committed plan). It specifies live
> cold-vs-warm exercises for the three eval axes the cumulative CRUD eval and the tactical traps do
> NOT exercise. No exercises are built or run this pass (spec only).

**Goal:** Design cheap, reproducible live tests for **C4** (memory adapts when a standard changes
over time / supersession), **C5** (memory surfaces recent history the agent should honor), **C6**
(memory drives emergent synthesis — a new integrative note from a cluster). Each test follows the
trap-harness pattern proven this session: a triggering condition, a cold arm and a warm arm, a
deterministic-or-judge check, a cold-confirm-first gate, and results in a labeled-criteria table.

**Context:** C1–C3 are measured (cumulative eval + the 5 tactical traps that flipped 0/25→25/25
cold→warm at opus). C4/C5/C6 each already have **scoring machinery + unit fixtures wired into
`validate.py`** — they are NOT yet wired as live model-running exercises. This plan closes that gap;
it does not reinvent the scorers.

## Existing machinery to build on (verified in code)

- **C4** — `dev/eval/cumulative/reversal_scorer.py::score_supersession(code, spec)` → `{follows_x,
  follows_x_prime, supersession_correct}`. Spec `reversal_spec.json`: X = "wrap errors with `%w`",
  X′ = "use bare sentinel errors (`var ErrX = errors.New(...)`), do NOT wrap". `supersession_correct
  = follows_x_prime and not follows_x`.
- **C5** — `recency_probe.py::score_recency_hit(payload, target)` → `{recent_channel_items,
  target_surfaced}`; `recent_channel_surfaced(payload, basename)`.
- **C6** — `synthesis_judge.py::judge_crystallization(vault, task, expected_z, ...)` →
  adversarial separate-model verdict `SYNTHESIS | NOT_SYNTHESIS` (defaults NOT_SYNTHESIS); fixtures
  `synthesis_fixtures/{fixture1..3, vault_with_cluster, vault_covered}`.

## Global constraints (load-bearing, from memory)

- **The signal must come from MEMORY, never the prompt.** The superseding standard (C4), the recent
  decision (C5), and the synthesis-enabling cluster (C6) live in the vault/chunk index — the task
  prompt must NOT state them. Otherwise the test measures prompt-following, not memory. This is the
  single most important design rule.
- **Cold-confirm first (saturation gate).** Before any warm spend, run the cold arm n=5: the check
  must reliably show the un-helped behavior. If cold already passes (the model does the right thing
  unaided), the axis is saturated on that case — drop/redesign it. (Per the opus oracle-saturation
  finding and the trap protocol.)
- **C5 recency must test recency, NOT relevance** (vault notes 42/44, 34): a recent planted item
  among *homogeneous old* distractors only proves the recency-band fires, not discrimination. Spread
  distractor ages (0.5d, 2d, 5d, 15d, 60d) and include a mid-age probe whose surfacing depends on
  recency, not cosine. Pure topic-cosine recall structurally cannot distinguish recent-from-corpus,
  so the test must isolate the recent channel.
- **Results in a labeled criteria table with units** — columns mapped to C1–C6, arms side by side,
  Δ column, axes-not-exercised marked.
- Spec only this pass; deterministic checks preferred (C4, C5-application) over judge (C6).

## Honest framing — these three are NOT of-a-kind

- **C4 is the flagship**: a deterministic check on the model's *code output* — directly comparable
  to C1–C3, and it measures real downstream task value (did the agent adopt the current standard?).
- **C5 is partly mechanical**: the recent channel surfaces newest chunks by construction; the test's
  real question is whether the agent *acts* on a surfaced-but-not-most-relevant recent item.
- **C6 tests capture quality, not task value**: it judges whether the `/recall` skill *writes a good
  integrative note* — a property of the skill's judgment, adversarially scored. A SYNTHESIS verdict
  must NOT be over-read as "memory made the model build better." Report it as what it is.

## C4 — Standards change over time (supersession) — FLAGSHIP

**Claim under test:** when a convention is established and later *changed*, memory carries the
**current** standard so the agent adopts X′ and drops X — without being told in the prompt.

**Condition:** the error-handling convention (anchors to the `wrapped-error` trap: cold opus
defaults to bare `return err`).
- **Arm A — memory has only the OLD note (X = "wrap with %w").** Recall surfaces X. Expect: agent
  wraps. `score_supersession` → `follows_x`.
- **Arm B — memory has X PLUS a newer superseding note (X′ = "use sentinels, X is superseded").**
  Recall surfaces both; recency-wins resolves to X′. Expect: agent uses sentinels, drops wrapping.
  `score_supersession` → `supersession_correct` (follows_x_prime and not follows_x).
- **Cold control** — no memory: agent uses its default (bare return err / neither). Confirms the
  signal isn't the prompt.

**Task prompt (no convention mentioned):** "Implement `Load(path string) ([]byte, error)` that reads
a file and returns its bytes, returning any error to the caller." (Same shape as `wrapped-error`.)

**Check (deterministic):** `reversal_scorer.score_supersession(generated_code, reversal_spec)`.
- C4 metric = **supersession rate**: fraction of Arm-B trials where `supersession_correct` is true.
- The flip that proves C4: Arm A `follows_x` → Arm B `supersession_correct`. Memory didn't just
  install a convention (C3) — it *reversed* an installed one when the standard changed.

**Cold-confirm / saturation gate:** Arm A must reliably produce `follows_x` (cold opus, given the
old note, wraps). If even Arm B's superseding behavior happens without the X′ note (model defaults
to sentinels), the case is saturated — pick a different X→X′ where X′ contradicts the model's prior.

**Cost:** trap-class, ~$0.30–0.50/trial warm (recall overhead). n=5 per arm × 3 arms.

## C5 — Recent history surfaced and honored

**Claim under test:** a decision made *recently* (in-session, not a crystallized lesson) surfaces in
recall's recent channel and the agent honors it — even when it isn't the most topically-relevant
item.

**Condition:** plant a recent chunk R stating an arbitrary in-session decision (e.g. "for this
project we settled on 2-space indentation and `// NOTE:` markers, decided just now"), ingested as
recent activity. Surround it with older distractor chunks on adjacent topics, **age-spread**
(0.5d/2d/5d/15d/60d) per the recency-eval lesson, with a mid-age probe.
- **Warm:** R is in the recent channel. `recent_channel_surfaced(payload, R)` = true. The build task
  is adjacent (so R is recent but not top-cosine). Expect: agent applies R's decision.
- **Cold / R-absent control:** no R → agent uses its default indentation/markers.

**Task prompt (decision not mentioned):** a small formatting-sensitive task where R's decision would
visibly apply (e.g. "add a short function to utils.go").

**Check (two-stage):**
1. **Surfaced (deterministic):** `score_recency_hit(recall_payload, R_basename).target_surfaced` —
   did the recent channel carry R? This isolates the recency *mechanism*.
2. **Honored (deterministic grep or light judge):** did the generated code follow R's decision
   (2-space indent, `// NOTE:` markers)? This is the *value* — agent acted on recent history.

**Saturation / validity gate:** confirm R surfaces via **recency, not cosine** — run a variant where
R is topically *distant* from the query; it must still surface in the recent channel. If R only
surfaces when it's also top-cosine, the test isn't measuring C5 (note 42/44 failure mode).

**Caveat:** C5's "honored" check is the real signal; "surfaced" is mostly mechanical. Report both
but weight honored.

## C6 — Emergent synthesis (compounding lessons)

**Claim under test:** when the vault holds a *cluster* of related notes that individually don't
state an integrative principle, recall crystallizes a NEW note Z that synthesizes them (not a
restatement) — and does NOT fire when the principle is already covered.

**Condition (fixtures exist):** `synthesis_fixtures/vault_with_cluster` (cluster present, no
integrative note) vs `vault_covered` (principle already noted).
- **Warm / cluster present:** agent runs `/recall`; expect it to crystallize Z. Judge:
  `judge_crystallization(...)` → SYNTHESIS (Z is integrative, not restatement).
- **Control / covered:** expect NO new synthesis note (MUST_NOT_FIRE) — guards against firing on
  everything.

**Check (adversarial judge):** `synthesis_judge.judge_crystallization` (separate-model, majority of
3, defaults NOT_SYNTHESIS). C6 metric = SYNTHESIS rate on cluster fixtures AND MUST_NOT_FIRE rate on
covered fixtures (both matter — a judge that always says SYNTHESIS is useless).

**Live gap to close:** today the judge runs on fixtures in `validate.py` (stubbed). The live test is
whether a real `/recall` invocation, against the cluster vault, actually *crystallizes* Z — i.e.,
fires `engram learn` at recall time. Measure: did recall write a new note? Then judge that note.

**Honest caveat (restated):** C6 measures the `/recall` skill's capture judgment, not downstream
build quality. Keep it labeled as such.
